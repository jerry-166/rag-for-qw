# 2026-04-10 工作日志

## RAG 检索架构重构 — Reranker 模块化

### 完成内容
1. **新建 `backend/services/reranker.py`** — 独立 Reranker 服务模块，包含：
   - `BaseReranker` 抽象基类（统一接口）
   - `LLMReranker` — 原 LLM Prompt 式实现（增强了解析容错：JSON + 正则降级）
   - `CrossEncoderReranker` — Cross-Encoder 实现（推荐生产使用）
   - `NoopReranker` — 空操作（调试/低延迟场景）
   - `get_reranker()` 工厂函数（全局单例 + 配置驱动切换）

2. **修改 `backend/config.py`** — 新增配置项：
   - `RERANKER_TYPE`: llm / cross_encoder / none
   - `RERANKER_MODEL`: 默认 BAAI/bge-reranker-v2-m3

3. **重构 `backend/api/search.py`**：
   - 删除内联的 `rerank_results()` 函数和 `rrf()` 函数
   - 改用 `services.reranker.get_reranker()` 获取独立实例
   - `rrf_fusion()` 现在返回 `(ids, scores_dict)` 元组，**修复了原来RRF分数重复计算的问题**
   - 提取辅助函数：`_build_metadata_filter`, `_build_es_filters`, `_es_search`, `_milvus_search`, `_enrich_results`, `_optional_rerank`
   - `QueryRequest` 新增 `use_rerank: bool = True` 参数，前端可控是否跳过精排

### 设计决策
- **Reranker 定位为独立服务模块**，不属于任何一个召回源，通过依赖注入方式按需调用
- Pipeline 顺序不变：多路并行召回 → RRF融合 → PG补全 → Rerank精排 → 返回
- 向后兼容：默认 RERANKER_TYPE=llm，行为与重构前一致

## 前端 Bug 修复（用户截图反馈）

### 修复内容（仅 `frontend/js/pages/search.js`）
1. **删除流水线提示条** — 用户标注"去掉"，已移除 `#pipeline-hint` 整个 DOM 块、`updatePipelineHint()` 方法及其所有调用
2. **修复分数展示逻辑** — 用户问"如何计算得到的相似度"
   - 旧：所有分数一律 `*100%` 显示，RRF 分被错误显示为百分比让人困惑
   - 新：区分两种分数类型：
     - **RRF 融合分**：直接显示原值如 `0.0333`，不转百分比；颜色分级阈值调整为 ≥0.05(高)/≥0.02(中)
     - **Rerank 精排分**：显示为百分比如 `92.5%`；阈值 ≥0.8(高)/≥0.4(中)
     - 卡片头部优先展示 rerank_score（如有），否则 rrfScore
     - hover tooltip + 展开详情同时展示双分数
3. **修复展开详情字段优先级** — 用户标注"完整内容应该展示chunk_text"
   - 旧：`result.content || result.chunk_text || ''`
   - 新：`result.chunk_text || result.content || ''`
