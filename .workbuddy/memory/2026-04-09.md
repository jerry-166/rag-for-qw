# 2026-04-09 工作日志

## 修复 pipeline.js 两个 UI 问题

### 问题1：Chunk 预览出现 markdown 层级假象
- 原因：`.chunk-preview` 直接截取 `chunk.content` 前80字符，其中含 `#` 标题符号
- 修复：新增 `_stripMarkdown()` 方法，在预览前剥离 `#`、`**`、`*`、链接等 markdown 符号

### 问题2：Step4 嵌入入库统计数据需点两次才显示
- 原因：`renderStep4()` 用模板字符串渲染时 `importResults` 还是 null，数据回来后没有回填 DOM
- 修复：
  1. 模板里统计卡片和时间线全部加 `id`，用加载动画占位
  2. 新增 `_fillImportStats()` 方法，在 `loadImportResults()` 拿到数据后立即调用，直接设置各 DOM 文本，无需重渲染整个页面

## 修复 generate 接口缓存分支状态遗漏

- 原因：`/api/process/generate/{file_id}` 缓存命中（已有子问题和摘要）时直接返回 200，但没把文档状态从 `chunk_done` 更新为 `generated`，导致后续 `/api/process/import` 检查 `status not in ["generated", "completed"]` 时返回 400
- 修复：缓存分支中补充 `db.update_document(file_id, status="generated")`

## 修复 Step4 嵌入入库重复调用 import API

- 原因：`nextStep()` 里先调了一次 `DocumentAPI.importToMilvus()`，然后 `renderStep4()` -> `loadImportResults()` 又调了一次，导致同一次点击 import 被执行两次，第一次结果丢失
- 修复：移除 `nextStep()` 中的重复 import 调用，改为由 `renderStep4` -> `loadImportResults` 统一处理；import 成功后自动标记 step4 为 done

## 升级删除文档/知识库接口 — 统一五层清理

### 原有问题
1. `database.py delete_document` 里用 `document_id`（数字ID）找 `temp/{id}.pdf`，但 temp 里存的是 UUID → 永远找不到
2. `file_path` 是相对路径（如 `1/2/uuid/original.pdf`），`os.path.exists()` 在错误目录下查找 → doc_storage 文件永远删不到
3. ES 索引没删（app.py delete_document 只删了 Milvus）
4. 功能重复：database.py 操作文件系统、app.py 也操作 Milvus/ES，职责不清

### 修复方案（职责分离）
- **`storage.py`**：新增 `delete_dir()` 方法，支持 Local 和 OSS 两种后端删除整个目录
- **`database.py delete_document`**：只负责 PostgreSQL 数据清理（sub_question → chunk_summary → document_chunk → document），不再碰文件系统
- **`app.py delete_document`**：统一调度 Milvus → ES → Storage（通过 file_path 反推文档目录）→ temp/output → Database
- **`app.py delete_knowledge_base`**：先查出所有文档，对每个文档执行五层清理，最后删除 knowledge_base 记录
- ES 删除时用文档实际 owner 的 `user_id`（而非当前操作者），确保 admin 删除时 routing 正确

## 修复 delete_document 的 temp/output 删除逻辑

- **根因**：`delete_document` 中 temp 清理用了 `str(file_id)`（数字ID如"5"）去匹配 `temp/{uuid}.pdf`，永远找不到；而 `delete_knowledge_base` 已从 `file_path` 反推 UUID，是正确的
- **修复**：统一用 `doc["file_path"]` 第3段（UUID）去定位 temp 和 output 文件，与 `delete_knowledge_base` 逻辑一致
- **关键知识点**：`document.id` 是自增整数，`file_path` 中嵌入的 `file_id` 是上传时生成的 UUID，两者不同！
